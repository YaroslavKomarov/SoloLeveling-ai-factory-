# uuid_generate_v4() not available in Supabase Cloud

**Date:** 2026-02-18
**Files:** supabase/migrations/001_initial_schema.sql
**Severity:** medium

## Problem

`npx supabase db push` fails with:
```
ERROR: function uuid_generate_v4() does not exist (SQLSTATE 42883)
```
Migration uses `uuid_generate_v4()` from the `uuid-ossp` extension, which Supabase Cloud does not allow creating via SQL (`CREATE EXTENSION` is restricted to the Dashboard).

## Root Cause

`uuid-ossp` is a non-default PostgreSQL extension. In Supabase Cloud,
extensions are managed by the platform — you cannot install them via
`CREATE EXTENSION` in migrations. The migration included
`create extension if not exists "uuid-ossp"` but this either fails
silently or doesn't expose `uuid_generate_v4()` in the `public` schema
search path, causing the subsequent table creation to fail.

PostgreSQL 13+ (the version Supabase uses) ships with `gen_random_uuid()`
as a built-in function — no extension required.

## Solution

Replaced all occurrences of `uuid_generate_v4()` → `gen_random_uuid()`
and removed the `uuid-ossp` extension line. The `vector` extension
(pgvector) is kept because Supabase explicitly supports it via Dashboard.

## Prevention

- Never use `uuid_generate_v4()` in Supabase migrations — always use `gen_random_uuid()`
- Only use extensions listed in Supabase Docs → Database → Extensions as supported
- `vector`, `pg_trgm`, `pg_stat_statements` are safe; `uuid-ossp` is not needed
