# Test assertion wrong: knowledge goal strategic count vs Ebbinghaus repetitions

**Date:** 2026-02-19 00:25
**Files:** src/test/tasks/spaced-repetition.test.ts
**Severity:** low

## Problem

Test `generateGoalPlan > knowledge-based goal: generates more strategic than regular tasks` failed with:
```
AssertionError: expected 9 to be greater than 14
```

The test counted total scheduled task *instances*, but regular tasks undergo Ebbinghaus spaced repetition — each regular task spec generates up to 7 scheduled occurrences across the 90-day window.

## Root Cause

The test used `{ regular: 1, strategic: 5 }` and `{ regular: 1, strategic: 4 }` expecting strategic (9) > regular (2). But the implementation generates Ebbinghaus repetitions for regular tasks: each spec produces up to 7 instances → 2 specs × 7 = 14 regular rows vs 9 strategic rows.

The test was written thinking in terms of "task specs per quest" without accounting for the multiplication effect of spaced repetition in the output count.

## Solution

Changed `regular: 1` to `regular: 0` for both quests in the knowledge-goal test case:
```typescript
tasksPerQuest: [
  { regular: 0, strategic: 5 },
  { regular: 0, strategic: 4 },
],
```
This makes strategic (9) definitively exceed regular (0), correctly capturing the intent that knowledge goals are strategic-heavy.

## Prevention

- When writing tests for task generation, remember that `regularTaskCount` is a *spec count*, not a *scheduled instance count* — each spec multiplies by up to 7 via Ebbinghaus intervals
- If the test intent is to verify the ratio of task types, use `regular: 0` for one type to get an unambiguous result, or count unique task specs rather than scheduled instances
- Alternative: add a helper that counts unique quest-level task entries rather than all expanded scheduled rows
